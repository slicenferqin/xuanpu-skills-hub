# Context Offload - 上下文卸载

将长输出卸载到文件系统，保持上下文窗口精简高效。

## 为什么需要上下文卸载？

| 问题 | 影响 |
|------|------|
| 长输出占用 tokens | 一次代码生成消耗 2000+ tokens |
| 上下文过长 | "中间迷失"现象，注意力分散 |
| 多轮对话退化 | 早期信息被"挤出"有效注意力范围 |

## 规则

### 必须卸载到文件

满足以下**任一**条件时：

1. **长度阈值**：输出超过 2000 字符
2. **显式请求**：用户说"保存到文件"、"save to file"
3. **多文件操作**：一次创建/修改 3+ 个文件
4. **完整文件内容**：展示完整文件（非片段）

### 必须输出到上下文

1. 用户明确要求"显示在上下文"、"show in context"
2. 输出少于 1000 字符
3. 简短解释、调试、问答

## 输出格式

### 卸载到文件时

1. 创建目录（如不存在）：`.claude-output/`
2. 生成文件名：`{YYYY-MM-DD}-{HH-mm}-{主题}.md`
3. 写入内容
4. 在上下文返回摘要：

```
## 已卸载到文件

**文件**: `.claude-output/{filename}`
**内容**: {1-2 句摘要}
**节省**: ~{字符数} 字符 (~{token数} tokens)

查看: `cat .claude-output/{filename}`
```

### 文件内容格式

```markdown
# {标题}

> Generated by Claude | {日期}
> Request: {请求简述}

{实际内容}
```

## 示例

### 示例 1：长代码生成

**用户**: "创建一个用户认证的 React 组件"

**执行**:
1. 预估输出 ~3000 字符 → 超过阈值
2. 写入 `.claude-output/2026-01-31-16-30-react-auth.md`
3. 返回摘要，节省 ~2500 字符

### 示例 2：短回答

**用户**: "let 和 const 有什么区别？"

**执行**: 直接输出（简短解释）

### 示例 3：用户覆盖

**用户**: "显示完整代码，不要保存文件"

**执行**: 输出到上下文（尊重用户偏好）
